Commands

Slash commands defined in .claude/commands/. Each has a single responsibility; sub-modes are listed when they exist.

/adr

Detect or author Architecture Decision Records. Six criteria define what qualifies; a two-question gate confirms.

  • /adr — Detection: scan since the last ADR and surface candidates.
  • /adr [topic] — Authoring: directly author an ADR for a known decision.

/audit

Full alignment audit across 12 invariant layers. Every check produces observed evidence — count, file:line, or diff. No "OK" without proof.

/day-end

Close the day: daily review entry, structural alignment check, commit, and sync to main. Handles the 05:00 boundary autonomously. No confirmation prompts.

/journal-chapter

Add or edit a chapter in the personal journal — the story behind how the Brand OS was built. Side project outside Brand OS scope; does not trigger devlog or roadmap updates.

/roadmap

Detect roadmap events or apply status changes. Both docs/ROADMAP.md and system/docs.html are always updated together.

  • /roadmap — Detection: surface events that need a status change.
  • /roadmap [action] — Update: apply a known transition directly.
  • /roadmap reorder — Reorder phases interactively in both files.

/session-end

Build a session summary, inject it into system/logbook.html, and commit. Also checks roadmap activity. Push happens via /day-end, not here.

/svg

Optimize and register one SVG icon — the single execution path for iconography. Validates anatomical name, aliases, category, functional mapping, active state, and each file write before proceeding.

  • Always one icon per call. Accepts inline SVG, a file path, or prompts for paste.

/syncmain

Push all committed work on main to remote. GitHub Pages updates automatically on push. Verifies a clean working tree first.

Rules

Non-negotiable constraints active in every session. Derived from CLAUDE.md and PROCESS.md.

Editing

  • str_replace only — Never rewrite a complete file when a section changed. Use section markers as anchor points.
  • One file per operation — A content update touches index.html only. A token update touches tokens.json only.
  • Never hand-edit brandOS-tokens.css — It is generated. Edit tokens.json, then run node scripts/build-tokens.js.
  • No <style> in index.html — All styles live in the CSS files. Existing inline styles on specific elements are intentional — don't add, don't remove.
  • Preserve section markers — Every <!-- SECTION: id --> and <!-- /SECTION: id --> must remain intact after any edit.

Content

  • content.md is source of truth — No text visible in index.html may exist without first being in brandOS-content.md. This includes labels, captions, and demo copy.
  • .pr is verbatim — The .pr paragraph is the first paragraph of the section, verbatim. Never condense, rewrite, or invent it.
  • PROCESS.md is mandatory — Before touching any file, identify the operation type and apply the corresponding protocol. No exceptions.

Naming & typography

  • Brand OS — Always with a space and capitalised initials. Never BrandOS, brandOS, or BRAND OS. Technical filenames (brandOS-*.css) are exempt.
  • IRON — Always bold and uppercase in body copy (<strong>IRON</strong> in HTML). In headings, heading weight provides the emphasis. Exempt in JS string contexts.
  • InsideBoard AI — Capital I and B, space, uppercase AI. Never "Insideboard AI" or "Inside Board AI" in copy. Deliberate all-caps graphic treatments are exempt.

Language

  • All files in English — Code, comments, commands, audits, ADRs, devlog, backlog, PROCESS.md, CLAUDE.md. Even when the chat is in French.
  • French is chat-only — French belongs to the live conversation, never to committed artifacts. Brand content is English-only; a French version would be an explicit, separate operation.

Structure

  • Bidirectional sync after structural ops — After adding, removing, or renaming a section or layer: every ### heading in content.md must have a matching SECTION comment in index.html, and vice versa.
  • JS block is frozen — The inline script at the bottom of index.html handles nav, register switcher, and density tweaks. Do not modify, move, or rewrite it.

Icons

  • One per /svg call — Never batch. Validate each gate before proceeding to the next.
  • Anatomical filenames — Names describe shape, never function (arrows-left-right.svg, not resize.svg). Functions live as semantic aliases in the registry.
  • Refuse outline-to-stroke — If a SVG is detected as vectorized fills (Illustrator export), alert and stop. Conversion is not reliably automatable.

Files

The project's file architecture — sources of truth, generated artifacts, and their relationships.

tokens.json Source of truth for all design tokens (W3C Design Tokens format).
brandOS-tokens.css Generated from tokens.json — never hand-edit.
scripts/build-tokens.js Generator: tokens.json → brandOS-tokens.css.
brandOS-components.css All component styles, layout, navigation, and responsive breakpoints.
index.html Pure HTML, named for GitHub Pages. Derived entirely from brandOS-content.md.
brandOS-content.md Content source of truth — maps 1:1 to HTML sections. No text enters index.html without existing here first.
PROCESS.md Full update protocols for every operation type. Mandatory reading before any edit.
docs/ Project documentation: ROADMAP, ADRs, devlog, backlog.
system/ Design system playground — consumes root CSS, never duplicates it.
.claude/commands/ Slash command definitions — one .md file per command.
