## Notable findings

- **B1 — journal.html inline style block (L8).** 420 lines of CSS had been written directly into `system/journal.html` as an inline `<style>` block — a rule violation that had accumulated silently since the journal page was built. The CSS was extracted to a `/* SYSTEM · journal */` block in `brandOS-components.css` and the inline block removed. The fix also required updating two CSS selectors that were targeting `h4` elements (`.terc h4`, `.ironh h4`) which needed to move to `h3` anyway for the heading hierarchy fix (M7).

- **B2 — PROCESS.md Update type 9 status gap.** The status mapping table for roadmap → docs.html sync only documented `done` and `active`. The `planned` and `onradar` values were already in use in `system/docs.html` but had no corresponding protocol documentation — any future phase update could have silently used the wrong CSS modifier. Added `planned` and `onradar` rows.

- **M7 — Heading level skips (L5.5).** Both `s15` (5 territory cards) and `s16` (IRON header) were using `h4` directly under `h2` with no intervening `h3`. CSS selectors had also been targeting `h4`, so both the HTML and the CSS selectors were corrected in the same operation.

- **M6 — Orphan file with illegal filename.** `assets/illustrations/*journal-ch3.png` had a literal asterisk in its filename — a filesystem artifact from a shell glob expansion gone wrong at some point. The file was not referenced anywhere (journal.html references `../journal/illustrations/journal-ch3.png` in a separate directory). Deleted.

- **M4 — Two missing devlog files.** 2026-04-28 (1 large foundation-sync commit) and 2026-04-29 (24 commits across 6+ sessions) had no devlog files despite logbook entries and significant activity. Both written retrospectively from git history.

- **M5 — 5 unreferenced logo/image files (open).** `assets/images/` contains `logo-insideboard-brandos.svg`, `logo-insideboard-ivory.svg`, `mark-insideboard-ivory.svg`, `logo-brandos.png`, `logo-brandos-v2.png` — all confirmed unreferenced via grep. Not deleted; awaiting user confirmation on whether these are intentional archives or safe to remove.

- **I1 — ADR-009 supersedes previous L9 finding.** The first audit (2026-04-27) flagged governance files on `main` as a violation and removed them. ADR-009 (single-branch workflow, adopted 2026-04-29) makes their presence deliberate. The layer now reads as clean.
